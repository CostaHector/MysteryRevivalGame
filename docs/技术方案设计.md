## 3. 现状与差距分析

当前代码采用 **“代码优先”** 架构（详见 `AGENTS.md`）：节点几乎全部在 C# `_Ready()` 里 `new` + `AddChild`，`.tscn` 极简。运行主链：

```
Main.tscn → Main._Ready() 造封面+相机+HUD
   → HUD 开始按钮 → Main.NewGame()
       → BornRoom(tscn, 含导航网格/门) + PlayerSprite + PlayerInteractDisplay + Backpack
```

**关键差距（决定实现顺序）：**

| 现状 | 差距 | 影响需求 |
|------|------|---------|
| `PlayerInteractDisplay` / `Backpack` 的格子是**纯视觉**：图标硬编码（slot0/1 写死红鬼烛），数量固定显示 `"0"` | **完全没有物品数据模型**，无法“消耗 / 拾取 / 移动 / 显示详情” | 3,4,5,6 |
| Hotbar(9) 与 Backpack(36) 各自独立构建，无共享数据 | 需要一个**统一的库存模型**并让两个 UI 都订阅刷新 | 3,4,5,6 |
| `PlayerSprite` 有 WASD + 点击门自动寻路，无“瞬移”“能力”概念 | 需新增鬼域状态、渲染、瞬移输入 | 1,2 |
| 无 Autoload 单例；跨节点靠 `GetParent<Main>().GetNode(...)` 字符串查找 | 库存需被 HUD/背包/世界物品多处访问，引入单例；玩家加入 group 便于查找 | 3,4,6 |
| `project.godot` 输入表仅 WASD/E/Esc | 缺 `[ ] = -` 动作 | 1,2 |
| 世界中无“可拾取物”节点类型 | 需新增 `WorldItem` | 4 |

> **实现铁律**：先落地 **第 5 章「核心数据模型」+ 单例**，再做 3/4/5/6；鬼域(1/2)与物品系统解耦，可并行。

### 4.1 新增/改动模块总览

```

[新增] Item/

[新增] WorldItem.cs        —— 场景中可右键拾取的物品（Area2D，含 PickupRange）
[新增] GhostDomain.cs      —— 鬼域渲染节点（Node2D，_Draw 画半透明红圆）

[改动] PlayerSprite.cs         —— 鬼域状态/瞬移/半径调节（需求 1,2）；加入 "player" group（供拾取测距）
[改动] PlayerInteractDisplay.cs —— 槽位从模型渲染（映射最后一行）；右键消耗选中槽（需求 3）
[改动] Backpack.cs             —— 槽位从模型渲染；tooltip（需求 5）；点击移动（需求 6）
[改动] Main.cs                 —— 初始化库存内容；在世界放置若干 WorldItem 供测试
[改动] project.godot           —— 新增 Autoload 与 4 个鬼域输入动作
```

### 4.2 数据与 UI 分离（核心原则）

```
                 ┌─────────────────────────────┐
                 │  GameState (Autoload 单例)   │
                 │    InventoryModel Inventory  │
                 └───────────────┬─────────────┘
                     SlotChanged(index) 信号
          ┌────────────────┬─────┴──────────┬───────────────┐
          ▼                ▼                ▼               ▼
 PlayerInteractDisplay  Backpack        WorldItem       PlayerSprite
 (Hotbar 映射 27..35)   (显示 0..35)    (拾取→AddItem)   (瞬移/能力)
 右键消耗选中槽          点击移动/悬停      右键+靠近触发     加入 player group
```

- **单一数据源**：所有物品状态只存于 `InventoryModel`；UI 只负责渲染 + 转发用户输入到模型方法。
- **单向刷新**：模型改变 → 发 `SlotChanged(index)` → 各 UI 局部重绘该格。杜绝 UI 之间直接读写。

### 4.3 Hotbar 与 Backpack 的槽位关系（已确认：共享库存，Hotbar = 最后一行）

采用 **Minecraft 式统一库存**，且 Hotbar 对应背包**最后一行**：

- `InventoryModel` 共 **36 槽**（索引 0–35），4×9 行主序（row-major）排布：
  - row0 = 0–8，row1 = 9–17，row2 = 18–26，**row3 = 27–35**。
- `Backpack` 显示全部 36 槽（索引 0–35）。
- `PlayerInteractDisplay` 的 Hotbar（9 格）镜像 **row3（索引 27–35）**：
  - Hotbar 槽位 `i`（0–8）↔ 库存索引 `27 + i`。
- 二者订阅同一模型：背包最后一行改动 → Hotbar 立即同步；Hotbar 消耗 → 背包最后一行立即同步。

> 该映射让需求 3（消耗 Hotbar 选中槽）、4（拾取入库）、6（背包内移动）共享同一份数据，无需跨 UI 同步逻辑。
> **统一索引约定**：所有槽位一律 **0-based**（`Slot0..Slot35`），与模型索引对齐。现有 `Backpack.CreateSlot(i+1)` 的 1-based 命名需改为 `CreateSlot(i)`（`i` 从 0 起）。

### 5.3 `ItemStack` —— 槽位内容

```csharp
// 一个槽位里的一堆物品。约定：库存中「空槽」直接用 null 表示，不用空 ItemStack。
public sealed class ItemStack {
    public ItemData Item;   // 非空
    public int Count;       // >= 1

    public ItemStack(ItemData item, int count) { Item = item; Count = count; }
    public ItemStack Clone() => new(Item, Count);
}
```

### 5.4 `InventoryModel` —— 库存数据（UI 无关）

```csharp
using Godot;
using System;

// 纯数据的库存模型：固定槽位数组，任何改动都通过 SlotChanged 通知 UI 局部刷新。
// 继承 RefCounted 以便使用 Godot 信号，同时不进入场景树。
public partial class InventoryModel : RefCounted {
    private readonly ItemStack[] _slots;
    public int SlotCount => _slots.Length;

    // 某个槽位内容发生变化（新增/减少/清空/移动）。参数为槽位索引。
    [Signal] public delegate void SlotChangedEventHandler(int index);

    public InventoryModel(int slotCount) { _slots = new ItemStack[slotCount]; }

    // 读取（可能为 null = 空槽）。UI 只读，不得直接改。
    public ItemStack GetSlot(int index) =>
        (index >= 0 && index < _slots.Length) ? _slots[index] : null;

    // 直接放置（主要用于初始化 / 测试）
    public void SetSlot(int index, ItemStack stack) {
        _slots[index] = stack;
        EmitSignal(SignalName.SlotChanged, index);
    }

    // 需求 4：加入物品，先堆叠到已有同类槽，再放入空槽。返回未能放下的剩余数量（0=全放下）。
    public int AddItem(ItemData item, int count) {
        // 1) 堆叠到已有同类且未满的槽
        for (int i = 0; i < _slots.Length && count > 0; i++) {
            var s = _slots[i];
            if (s != null && s.Item == item && s.Count < item.MaxStack) {
                int add = Math.Min(count, item.MaxStack - s.Count);
                s.Count += add; count -= add;
                EmitSignal(SignalName.SlotChanged, i);
            }
        }
        // 2) 放入空槽
        for (int i = 0; i < _slots.Length && count > 0; i++) {
            if (_slots[i] == null) {
                int add = Math.Min(count, item.MaxStack);
                _slots[i] = new ItemStack(item, add); count -= add;
                EmitSignal(SignalName.SlotChanged, i);
            }
        }
        return count; // >0 表示背包满，未放下的数量
    }

    // 需求 3：消耗指定槽位 count 个，返回实际消耗数。数量归零则清空该槽。
    public int ConsumeAt(int index, int count) {
        var s = GetSlot(index);
        if (s == null || count <= 0) return 0;
        int consumed = Math.Min(count, s.Count);
        s.Count -= consumed;
        if (s.Count <= 0) _slots[index] = null;
        EmitSignal(SignalName.SlotChanged, index);
        return consumed;
    }

    // 需求 6：把 from 槽移到 to 槽。约定：仅当 to 为空槽时执行“移动”。
    // 目标非空时不处理（由 UI 层改为“改选 B”，见 6.5）。
    public void Move(int from, int to) {
        if (from == to) return;
        if (_slots[to] != null) return; // 目标非空 → 不移动
        _slots[to] = _slots[from];
        _slots[from] = null;
        EmitSignal(SignalName.SlotChanged, from);
        EmitSignal(SignalName.SlotChanged, to);
    }

    // 预留：非空↔非空交换（当前需求未用，后续如需“拖拽交换”可启用）
    public void Swap(int a, int b) {
        (_slots[a], _slots[b]) = (_slots[b], _slots[a]);
        EmitSignal(SignalName.SlotChanged, a);
        EmitSignal(SignalName.SlotChanged, b);
    }
}
```

### 5.5 `GameState` —— Autoload 单例（含 Hotbar 映射）

```csharp
using Godot;

// 全局游戏状态单例（Autoload）。当前只持有库存，后续可扩展：复苏度、幸存者列表等。
public partial class GameState : Node {
    public static GameState Instance { get; private set; }

    public const int INVENTORY_SLOTS = 36;                    // 背包 4×9
    public const int HOTBAR_SIZE     = 9;                     // Hotbar 宽度
    public const int HOTBAR_START    = INVENTORY_SLOTS - HOTBAR_SIZE; // = 27，Hotbar 映射到最后一行

    public InventoryModel Inventory { get; private set; }

    public override void _EnterTree() {
        Instance = this;
        Inventory = new InventoryModel(INVENTORY_SLOTS);
    }

    // Hotbar 槽位(0..8) → 背包库存索引(27..35)
    public int HotbarToInventory(int hotbarIndex) => HOTBAR_START + hotbarIndex;
    // 背包库存索引 → Hotbar 槽位；不在 Hotbar 区间返回 -1
    public int InventoryToHotbar(int invIndex) =>
        (invIndex >= HOTBAR_START && invIndex < INVENTORY_SLOTS) ? invIndex - HOTBAR_START : -1;
}
```

在 `project.godot` 注册 Autoload（新增段）：

```ini

```

> 之后任意脚本用 `GameState.Instance.Inventory` 访问库存，无需 `GetParent<Main>()` 字符串查找。

## 6. 功能详细设计

### 6.1 需求 1 & 2：鬼域能力（展开 / 收起 / 缩放 / 瞬移）

#### 6.1.1 形状（已确认：整圆）

鬼域为**以角色为圆心的完整圆**，半透明红色填充，默认半径 400，范围内可无视导航直接瞬移。判定 = 目标点到角色的欧氏距离 ≤ 当前半径。

#### 6.1.2 渲染节点 `GhostDomain`

半透明红圆需跟随角色、且**不受 `PlayerSprite.Scale=0.5` 影响**（否则半径会被缩放）。因此 `GhostDomain` 作为**世界节点**（挂在玩家的父节点即当前房间/Main 下），每帧同步到玩家全局坐标：

```csharp
using Godot;

// 鬼域可视化：半透明红色整圆，半径由外部设置。放在世界空间（非 PlayerSprite 子节点），避免被玩家缩放影响。
public partial class GhostDomain : Node2D {
    private float _radius = 400f;
    public float Radius {
        get => _radius;
        set { _radius = value; QueueRedraw(); } // 半径变了要重画
    }

    private static readonly Color FillColor = new(1f, 0f, 0f, 0.25f); // 半透明红
    private static readonly Color EdgeColor = new(1f, 0.2f, 0.2f, 0.6f);

    public override void _Ready() {
        Name = "GhostDomain";
```

#### 6.1.4 输入处理（键盘 + 瞬移点击）

在 `PlayerSprite._UnhandledInput` 中处理（新增该方法）：

```csharp
public override void _UnhandledInput(InputEvent @event) {
    // —— 需求 2：鬼域键盘操作 ——
    if (@event.IsActionPressed("ghost_domain_expand"))  { ExpandGhostDomain();  GetViewport().SetInputAsHandled(); return; }
    if (@event.IsActionPressed("ghost_domain_collapse")){ CollapseGhostDomain();GetViewport().SetInputAsHandled(); return; }
    if (@event.IsActionPressed("ghost_domain_enlarge")) { ChangeGhostDomainRadius(+DOMAIN_RADIUS_STEP); GetViewport().SetInputAsHandled(); return; }
    if (@event.IsActionPressed("ghost_domain_shrink"))  { ChangeGhostDomainRadius(-DOMAIN_RADIUS_STEP); GetViewport().SetInputAsHandled(); return; }

    // —— 需求 1：鬼域内左键瞬移 ——
    if (_ghostDomainActive && CanMove
        && @event is InputEventMouseButton mb && mb.Pressed && mb.ButtonIndex == MouseButton.Left) {
        Vector2 target = GetGlobalMousePosition();
        if (IsInsideGhostDomain(target)) {
            SetNavigateFinished();     // 打断可能正在进行的自动寻路
            Position = target;         // 无视导航直接瞬移
            GetViewport().SetInputAsHandled();
        }
    }
}

// 目标点是否在鬼域圆内（整圆：欧氏距离 ≤ 半径）
private bool IsInsideGhostDomain(Vector2 worldPoint) =>
    GlobalPosition.DistanceTo(worldPoint) <= _ghostDomainRadius;
```

**交互流程（瞬移）**：
```
按 [ → 鬼域展开(红圆显示) → 左键点圆内某点
  → 距离<=半径? → 是：Position 直接=点，跳过 IsWalkable/导航；否：忽略
按 = / - → 半径±100（clamp 100~1000），红圆实时变化
按 ] → 鬼域收起，左键恢复无瞬移
```

**边界情况**：
- 背包打开（`CanMove=false`）时禁止瞬移；但红圆仍跟随显示（符合“非暂停”设定）。
- 瞬移点在圆外：忽略（不移动、不报错）。
- 瞬移会打断自动寻路：先 `SetNavigateFinished()` 再赋值 `Position`。
- 与需求 4 的右键拾取不冲突：**瞬移用左键，拾取/使用用右键**。

### 6.2 需求 3：右键消耗选中槽物品（`PlayerInteractDisplay`）

> Hotbar 槽位 `_selectedSlot`（0–8）对应库存索引 `GameState.Instance.HotbarToInventory(_selectedSlot)`（27–35）。以下读写一律走该映射。

#### 6.2.1 槽位改为由模型驱动

新增 `RefreshSlot(hotbarIndex)`，订阅 `SlotChanged`，并在 `_Ready` 初次全量刷新。删除硬编码图标。建议 `CreateSlot` 时把 `Icon{i}`/`Count{i}` 引用存入 `List<TextureRect>` / `List<Label>`，避免每次字符串查找。

```csharp
public override void _Ready() {
    Name = "PlayerInteractDisplay";
    BuilderTopStatusHotBar();
    BuildBottomHotbar();
    MoveSelectorTo(_selectedSlot);

    // 订阅库存变化
    GameState.Instance.Inventory.SlotChanged += OnInventorySlotChanged;
    for (int i = 0; i < SLOT_COUNT; i++) RefreshSlot(i); // SLOT_COUNT = 9
}

public override void _ExitTree() {
    // 防止场景切换后悬挂回调
    if (GameState.Instance?.Inventory != null)
        GameState.Instance.Inventory.SlotChanged -= OnInventorySlotChanged;
}

// 库存某格变了：仅当它落在 Hotbar 区间(27..35)时刷新对应 Hotbar 格
private void OnInventorySlotChanged(int invIndex) {
    int hotbar = GameState.Instance.InventoryToHotbar(invIndex);
    if (hotbar >= 0) RefreshSlot(hotbar);
}

// 从模型读取该 Hotbar 槽对应的库存内容，更新图标 + 数量文本
private void RefreshSlot(int hotbarIndex) {
    int invIndex = GameState.Instance.HotbarToInventory(hotbarIndex);
    var stack = GameState.Instance.Inventory.GetSlot(invIndex);
    var icon  = _icons[hotbarIndex];   // CreateSlot 时缓存的引用
    var count = _counts[hotbarIndex];
    if (stack == null) { icon.Texture = null; count.Text = ""; }
    else { icon.Texture = stack.Item.GetIcon(); count.Text = stack.Count > 1 ? stack.Count.ToString() : ""; }
}
```

#### 6.2.2 右键消耗

扩展现有 `_UnhandledInput`（当前只处理滚轮）：

```csharp
public override void _UnhandledInput(InputEvent @event) {
    if (@event is InputEventMouseButton mouseBtn && mouseBtn.Pressed) {
        // 滚轮切换选中槽（原有逻辑保留）...

        // 需求 3：右键消耗当前选中槽 1 个
        if (mouseBtn.ButtonIndex == MouseButton.Right) {
            int invIndex = GameState.Instance.HotbarToInventory(_selectedSlot);
            var stack = GameState.Instance.Inventory.GetSlot(invIndex); // 记住消耗前的物品定义
            int consumed = GameState.Instance.Inventory.ConsumeAt(invIndex, 1);
            if (consumed > 0) {
                GetViewport().SetInputAsHandled();
                // 决策 6：本期仅扣数量；使用效果走预留分发器（当前为空实现）
                ItemUseHandler.OnUsed(stack?.Item, FindPlayer());
            }
        }
    }
}

// 便捷查找玩家（用于把使用者传给效果处理器）
private PlayerSprite FindPlayer() =>
    GetTree().GetFirstNodeInGroup("player") as PlayerSprite;
```

**与需求 4 的右键冲突解决（关键）**：利用 Godot 输入传播顺序——
```
世界中的 WorldItem(Area2D) 的 InputEvent 先接收右键 → 命中且在拾取距离内 → SetInputAsHandled()（拾取，需求4）
   ↓ 未命中任何 WorldItem，或命中但太远未 handle
事件继续冒泡到 _UnhandledInput → 此处消耗选中槽（需求3）
```
即：**右键点在“够近的可拾取物”上 = 拾取；点在空地/其它 = 消耗选中槽**。传播顺序天然分流。

**边界情况**：选中空槽右键 → `ConsumeAt` 返回 0，不 handle，无副作用。

### 6.3 需求 4：右键拾取世界物品到背包（需靠近）

#### 6.3.1 `WorldItem` 节点（含拾取距离判定 —— 决策 2）

```csharp
using Godot;

// 场景中可被右键拾取的物品。用 Area2D 接收鼠标点击（先于 _UnhandledInput）。
// 决策 2：玩家与物品距离 ≤ PickupRange 才能拾取，否则忽略（事件不 handle，冒泡给消耗逻辑）。
public partial class WorldItem : Area2D {
    [Export] public string ItemId = "red_ghost_candle";
    [Export] public int Count = 1;
    [Export] public float PickupRange = 120f; // 允许拾取的最大距离（像素，世界坐标）

    public override void _Ready() {
        var data = ItemDatabase.Get(ItemId);
        var sprite = new Sprite2D { Texture = data.GetIcon() };
        AddChild(sprite);
        // CollisionShape2D 建议在 tscn 里配好，或此处按图标尺寸动态生成，提供点击范围
        InputPickable = true;
        InputEvent += OnInputEvent;
        MouseEntered += () => Input.SetDefaultCursorShape(Input.CursorShape.PointingHand);
        MouseExited  += () => Input.SetDefaultCursorShape(Input.CursorShape.Arrow);
    }

    private void OnInputEvent(Node viewport, InputEvent @event, long shapeIdx) {
        if (@event is InputEventMouseButton mb && mb.Pressed && mb.ButtonIndex == MouseButton.Right) {
            // 决策 2：距离判定 —— 太远则不拾取，也不 handle（让事件冒泡到“消耗选中槽”）
            var player = GetTree().GetFirstNodeInGroup("player") as Node2D;
            if (player == null || player.GlobalPosition.DistanceTo(GlobalPosition) > PickupRange) {
                // TODO: 可在此提示“距离太远”，本期静默忽略
                return;
            }

            var data = ItemDatabase.Get(ItemId);
            int remain = GameState.Instance.Inventory.AddItem(data, Count);
            GetViewport().SetInputAsHandled(); // 阻止事件继续冒泡到“消耗选中槽”
            if (remain == 0) QueueFree();       // 全部拾取 → 移除世界物品
            else Count = remain;                // 背包满 → 保留剩余
        }
    }
}
```

### 6.4 需求 5：背包格子悬停 tooltip（空槽不显示 —— 决策 3）

用 Godot 内建 `Control.TooltipText`：给格子 `Panel` 设文本，悬停约 0.5s 自动弹出系统 tooltip。**文本随物品变化，空槽清空文本即不显示**。因此 tooltip 文本在**背包的 `RefreshSlot`** 中一并设置：

### 6.5 需求 6：背包内点击移动物品（`Backpack`）

#### 6.5.1 状态机（含“再点 A 取消” —— 决策 4）

```csharp
private int _pendingSlot = -1; // 已选中、待移动的源格子；-1 表示当前无选中
```

规则（严格对应需求 + 决策 4）：

| 当前状态 | 点击的格子 | 行为 |
|---------|-----------|------|
| 无选中(`-1`) | 空格 | 忽略（没有可移动的物品） |
| 无选中(`-1`) | 非空格 A | 选中 A：`_pendingSlot=A`，高亮 |
| 已选中 A | **同一格 A** | **取消选中**（决策 4），清除高亮 |
| 已选中 A | **空格** B | **移动** A→B（`Inventory.Move`），清除选中与高亮 |
| 已选中 A | **非空格** B | **忽略 A**，改选 B：`_pendingSlot=B`，高亮切到 B |

```csharp
private void OnSlotClicked(int index) {
    var inv = GameState.Instance.Inventory;
    bool clickedHasItem = inv.GetSlot(index) != null;

    if (_pendingSlot == -1) {
        // 无选中：只有点到“非空格”才选中；点空格忽略
        if (clickedHasItem) SetPending(index);
        return;
    }

    // 已有选中 A = _pendingSlot
    if (index == _pendingSlot) { ClearPending(); return; } // 决策 4：再点自己 = 取消

    if (clickedHasItem) {
        // 需求 6：先 A 再点非空 B → 忽略 A，直接改选 B
        SetPending(index);
    } else {
        // 目标为空格 → 执行移动 A → B
        inv.Move(_pendingSlot, index);
        ClearPending();
    }
}

private void SetPending(int index) {
    if (_pendingSlot != -1) SetHighlight(_pendingSlot, false);
    _pendingSlot = index;
    SetHighlight(index, true);
}
private void ClearPending() {
    if (_pendingSlot != -1) SetHighlight(_pendingSlot, false);
    _pendingSlot = -1;
}
```

> 高亮 `SetHighlight(index, on)` 复用 `PlayerInteractDisplay` 里 `StyleBoxFlat` 边框选中的思路：在背包 `CreateSlot` 里给每个格子加一个可切换的 `StyleBoxFlat` 边框并存入 `List<StyleBoxFlat>`，切换其 `BorderColor/BorderWidth`。

#### 6.5.2 让格子可被点击

`Panel` 默认不发点击信号。给每个格子连 `GuiInput`：

```csharp
// CreateSlot(index) 内（index 0-based）
slot.MouseFilter = Control.MouseFilterEnum.Stop;
int captured = index; // 闭包捕获当前索引
slot.GuiInput += (InputEvent e) => {
    if (e is InputEventMouseButton mb && mb.Pressed && mb.ButtonIndex == MouseButton.Left) {
        OnSlotClicked(captured);
        AcceptEvent();
    }
};
```

#### 6.5.3 背包也要订阅库存刷新

背包同样订阅 `SlotChanged` 并有自己的 `RefreshSlot(index)`（渲染图标/数量 + 6.4 的 tooltip 文本）。`_Ready` 全量刷新，`_ExitTree` 退订。

**边界情况**：
- 背包关闭时 `_pendingSlot` 必须重置：在 `Close()` 里调 `ClearPending()`，避免下次打开残留高亮。
- 移动到的目标若落在 Hotbar 区间（27–35），Hotbar 会通过 `SlotChanged` 自动刷新（统一库存的红利）。
- 所有索引 0-based，与模型一致（务必先完成 4.3 / 5.x 的索引统一）。
