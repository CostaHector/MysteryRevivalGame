## ADDED Requirements

### Requirement: 副本成功结算
满足有效出口条件后，系统MUST锁定游戏输入、停止当前威胁并显示成功结算。

#### Scenario: 成功离开鬼域
- **WHEN** 玩家通过有效出口完成副本
- **THEN** 结算界面显示完成状态和本次关键结果

### Requirement: 结算操作
成功结算MUST提供重新游玩首个副本和返回标题界面的操作。

#### Scenario: 重新游玩
- **WHEN** 玩家在结算界面选择重新游玩
- **THEN** 系统清空本次副本状态并从出生房间初始阶段开始

#### Scenario: 返回标题
- **WHEN** 玩家选择返回标题
- **THEN** 系统卸载游戏地点和常驻玩法 UI，并重新显示标题界面

### Requirement: 结算与失败互斥
同一次副本运行中成功结算和失败界面MUST互斥，首先完成的终局状态MUST阻止另一状态出现。

#### Scenario: 出口与袭击同帧发生
- **WHEN** 有效出口完成与厉鬼袭击在同一帧提交
- **THEN** 系统按已定义的终局仲裁规则只进入一个终局状态
