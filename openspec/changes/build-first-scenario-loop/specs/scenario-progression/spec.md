## ADDED Requirements

### Requirement: 阶段式副本进度
系统MUST以明确阶段管理首个副本，每个阶段包含进入动作、目标、完成条件和唯一后继阶段。

#### Scenario: 完成当前阶段
- **WHEN** 当前阶段的全部必需目标完成
- **THEN** 系统执行阶段退出动作并进入配置的下一阶段

### Requirement: 事件驱动目标更新
副本目标MUST由领域事件更新，包括地点进入、物品拾取、调查完成、厉鬼压制和出口交互。

#### Scenario: 拾取关键手机
- **WHEN** 系统收到关键手机已拾取事件
- **THEN** 对应目标标记完成且目标 UI 更新

### Requirement: 世界标记持久于地点重载
本次游戏会话内，已拾取物品、已调查对象和已解锁出口MUST通过世界标记保持状态。

#### Scenario: 返回已探索地点
- **WHEN** 玩家重新进入已拾取关键物品的地点
- **THEN** 该一次性世界物品不得重新生成

### Requirement: 阶段检查点
系统MUST在配置的阶段入口创建可供失败重开的内存检查点。

#### Scenario: 进入追猎阶段
- **WHEN** 副本进入追猎阶段
- **THEN** 系统保存当前地点、出生点、库存、玩家状态和世界标记快照
