## ADDED Requirements

### Requirement: 统一交互描述
每个世界可交互对象MUST提供提示文本、触发按钮、交互距离、交互位置、可用性检查和执行结果。

#### Scenario: 悬停可用对象
- **WHEN** 鼠标进入当前可用的世界交互对象
- **THEN** 系统显示该对象提供的提示并使用可交互鼠标反馈

### Requirement: 自动接近后执行
需要靠近的交互请求MUST先让玩家导航到对象交互位置，并在到达后重新校验再执行。

#### Scenario: 点击远处的门
- **WHEN** 玩家点击距离外的门且存在可达路径
- **THEN** 玩家自动移动到门的交互位置并在到达后执行开门行为

#### Scenario: 到达前对象失效
- **WHEN** 玩家移动期间目标对象被释放或变为不可用
- **THEN** 系统取消待执行交互且不抛出错误

### Requirement: 单一待执行目标
系统MUST确保同一时间最多存在一个待执行世界交互，新请求MUST取消或替换旧请求。

#### Scenario: 行走中点击另一对象
- **WHEN** 玩家正走向对象 A 时点击对象 B
- **THEN** 系统取消 A 的待执行行为并改为处理 B

### Requirement: 输入传播优先级
被 UI 或世界交互接受的鼠标事件MUST停止传播，避免同一次点击触发瞬移或快捷栏使用。

#### Scenario: 右键拾取物品
- **WHEN** 世界物品接受右键拾取请求
- **THEN** 本次右键不得继续触发当前快捷栏物品使用
