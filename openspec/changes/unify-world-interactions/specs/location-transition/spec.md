## ADDED Requirements

### Requirement: 按地点标识切换场景
系统MUST通过地点 ID 加载目标地点，卸载当前地点，同时保留玩家和常驻 HUD 节点。

#### Scenario: 从出生房间进入废弃老宅
- **WHEN** 门交互请求切换到废弃老宅地点
- **THEN** 出生房间被卸载、废弃老宅被加载且玩家与 HUD 保持存在

### Requirement: 按出生点标识定位玩家
每次地点切换MUST使用目标地点中的出生点 ID 定位玩家，而不是依赖硬编码坐标。

#### Scenario: 使用前门入口
- **WHEN** 玩家通过出生房间前门进入废弃老宅并指定 `front-door-entry`
- **THEN** 玩家出现在废弃老宅对应出生点

### Requirement: 转场期间锁定输入
地点卸载、加载和玩家定位期间，系统MUST锁定移动和世界交互，并在目标地点准备完成后恢复。

#### Scenario: 连续点击出口
- **WHEN** 地点切换尚未完成时玩家再次点击出口
- **THEN** 系统忽略重复请求且只加载一个目标地点实例

### Requirement: 无效地点安全失败
当地点 ID 或出生点 ID 无效时，系统MUST保留当前可玩状态并报告明确错误。

#### Scenario: 出生点不存在
- **WHEN** 目标地点成功加载但指定出生点不存在
- **THEN** 系统回滚或使用明确配置的默认出生点，并记录错误
