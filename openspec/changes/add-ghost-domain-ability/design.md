## Context

玩家目前是 `Sprite2D`，同时承担移动和导航逻辑。鬼域需要跟随玩家、响应输入、改变范围并执行无视导航的瞬移。若直接把视觉、输入和规则全部加入 `PlayerSprite`，会妨碍后续复苏度、压制和场景限制扩展。

## Goals / Non-Goals

**Goals:**
- 提供独立、可查询的鬼域能力状态和瞬移 API。
- 提供跟随玩家的圆形半透明视觉反馈。
- 与背包移动锁定和未来厉鬼压制接口兼容。

**Non-Goals:**
- 不实现完整复苏度数值、鬼眼动画、伤害或多层鬼域对抗。
- 不允许鬼域瞬移跨越地点或触发场景出口。

## Decisions

1. **规则组件与表现组件分离。** `GhostDomainAbility` 管理状态和判定，`GhostDomainView` 只绘制圆形并响应信号。
2. **鬼域作为玩家子节点。** 使用本地原点自然跟随玩家，避免每帧同步世界坐标。
3. **瞬移使用全局鼠标坐标和圆形距离判定。** 成功时直接设置玩家全局位置，不查询导航网格，符合“域内无视地形”的能力定义。
4. **输入由能力组件集中处理。** 展开、收起、缩放和瞬移共享同一状态机，避免与普通点击寻路同时生效。
5. **提供策略钩子。** `CanActivate`、`CanTeleport` 和消耗回调允许未来环境、复苏度或剧情禁用能力。

## Risks / Trade-offs

- [瞬移进入不可见或不可交互区域] → 地点可提供可选禁入区域查询；初版至少限制窗口内和鬼域半径内。
- [左键与门交互冲突] → 可交互对象优先处理点击；只有未处理左键才尝试瞬移。
- [圆形绘制遮挡场景] → 使用低透明度、负向或受控 ZIndex，并禁止鼠标拦截。
- [能力输入与背包冲突] → `CanMove` 为 false 或 UI 已处理输入时拒绝能力操作。

## Migration Plan

新增输入动作和玩家子组件，默认鬼域收起，不改变现有移动行为。出现问题时可禁用组件而不影响导航移动。
