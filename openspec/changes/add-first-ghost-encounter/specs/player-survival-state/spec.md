## ADDED Requirements

### Requirement: 玩家生存状态模型
系统MUST以独立模型保存身体状态、精神压力、复苏度和死亡状态，并为每类变化发出通知。

#### Scenario: 精神压力增加
- **WHEN** 遭遇系统增加玩家精神压力
- **THEN** 模型保存新值并发出精神状态变更通知

### Requirement: 状态值边界
数值状态MUST限制在配置范围内，死亡状态一旦进入MUST保持到遭遇重置。

#### Scenario: 复苏度超过上限
- **WHEN** 复苏度增加量使结果超过最大值
- **THEN** 模型将其限制为最大值并触发对应失败规则

### Requirement: HUD 显示真实状态
顶部 HUD MUST显示玩家状态模型和当前环境提供的数据，不得继续使用固定占位文本。

#### Scenario: 进入黑暗鬼域
- **WHEN** 环境状态变为黑暗鬼域且可视范围降低
- **THEN** HUD 更新环境和可视范围文本

### Requirement: 死亡输入锁定
玩家死亡后系统MUST停止移动、瞬移、物品使用和世界交互。

#### Scenario: 死亡后移动
- **WHEN** 玩家已死亡并按下移动输入
- **THEN** 玩家位置保持不变
